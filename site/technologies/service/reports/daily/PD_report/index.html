    1. Hypercare Report¶

    Hypercare Report is a report for daily operations standup meeting which has the details of previous day highlights of all the teams for Enterprise Account. Most of the contents are provided by respective team’s/Squads.

    Topics covered in HC report

    • Incident Management
    • Change Management
    • Datastore Optimization Action Plan
    • Backup Status Report
    • Capacity Report
    • Cloud Infra Availability Report
    • Hotspots for each region covered by respective regional SDM
    • On call Rota (Enterprise and Enterprise). Sample of the HC report- which covers both for PiaaS and InfraIT

    2. Backup Report¶

    Steps to create the report

    Step 1: Use the file named “ Backup Daily Report_new_dd-mm-yyyy

    Step 2: Rename the file with today's date.

    Step 3: Required to update 3 tabs – Chart, OS and DB. Only OS and DB tab are updated, Chart tab will be automatically filled.

    Step 4: Go to the OS tab and clear the contents from A to O excluding heading.

    Step 5: Go to the DB tab and clear the contents from A to R excluding heading

    Step 6: Required automated reports are available on share folder location as given below https://enterprise.sharepoint.com/sites/EnterpriseReportdelivery/SitePages/ProjectHome.aspx?OR=Teams-HL&CT=1661402483534&clickparams=eyJBcHBOYW1lIjoiVGVhbXMtRGVza3RvcCIsIkFwcFZlcnNpb24iOiIyNy8yMjA3MzEwMTAwNSIsIkhhc0ZlZGVyYXRlZFVzZXIiOmZhbHNlfQ%3D%3D

    Step 7: Go the folder eg – AT – go the respective year xxxx month XX and respective day for which the report to be updated.

    Step 8: Go the OS backup tab, and copy from b to p excluding the heading and paste in to the OS tab of backup report file.

    Step 9: Go the DB backup tab, and copy from b to s excluding the heading and paste in to the DS tab of backup report file.

    Step 10: Repeat the same for all other regions ME, NE, NA and SE and paste them one below the other in respective tabs.

    Step 11: Once the pasting is done and sort by host name for both OS and DB Tab

    Step 12: The automated report will be available by 6:00 Pm everyday for next day’s report. Previous day data is always captured for next day report.

    Step 13: This report should be sent to backup team to update the 3rd table -IBM data.

    Step 14: Backup team will update the third one and send back to update the HC report and the final report is viewed as below

    4. Datastore Report¶

    Steps to create the report

    Step 1: Source data is received from cloud team via email

    Step 2: Download and open the file, delete columns B, C, E and I and apply filters, Datastore column , search for cache, and delete all the rows corresponding to it

    Step 3: % should be calculating on the last column = Utilized in GB/Capacity in GB.

    Step 4: Filter the % column for <80% and delete.

    Step 5: Then open data store template

    Step 6: It will have 3 tabs, Previous day, Today and ME-Open paas

    Step 7: Copy from today from a to J column and paste it on Previous day tab

    Step 8: From the automated file, filter for cloud group starting with series BNE, copy and paste the data in ME openPaas tab.

    Step 9: Rest of the data in Today’s tab.

    Step 10: Both Today’s data and ME-Open Paas data is shared in Hypercare report and sent to Cloud team Report looks as below –

    8. PIAAS I2BCK Report¶

    Steps to create the report

    Step 1: Working report Name - Infra_I2BCK_DLY_audit_logs - VM and DB View DD-Month– Update the date for which report is prepared

    Step 2: Share point link for automated report: https://enterprise.sharepoint.com/sites/EnterpriseReportdelivery/SitePages/ProjectHome.aspx

    Step 3: Path to download the automated report - Enterprise Report delivery - EnterpriseREPORTDELIVERY\PIAAS\REPORTS\I2BCK\OPERATIONAL

    Step 4: For NA we use N-1 report for other regions we should use same day report

    Step 5: Go to DB View & VM View

    Step 6: Copy the data from F to S Colum

    Step 7: Paste the date from E Colum.

    Step 8: Delete the data in in S Colum.

    Step 9: In S Colum enter the date for which we are preparing the report

    Step 10: Validate the DB Details in DB View in working report with the automated report.

    Step 11: To validate download the automated report from share point link. For the region we are working on.

    Step 12: In automated report put a vlookup formula in L Colum

    =Vlookup(Select B2 Colum in automated report, Select complete A Colum of DB view in working report,1,0)

    Step 13: Drag and drop formula till the end of the sheet.

    Step 14: Apply filler on and Select N/A in L Colum.

    Step 15: In K column Type if there is DB.

    Step 16: If there is no DB no action required. If we find DB we’ll have to select that and should copy the Node Name in B column and should paste it in DB view working sheet A column at the end of the sheet.

    Step 17: Validate the VM Details in VM View in working report with the automated report.

    Step 18: In automated report put a vlookup formula in L Colum

    =Vlookup(Select D2 Colum in automated report, Select complete A Colum of VM view in working report,1,0)

    Step 19: Drag the formula till the end of the sheet.

    Step 20: Apply filler on and Select N/A in L Colum.

    Step 21: In K column “Type” if there is VM.

    Step 22: If there is no VM no action required. If we find VM we’ll have to select that and should copy the Filespace Name in D column and should paste it in DB view working sheet A column at the end of the sheet.

    Step 23: Updating replication status on working sheet. Replication successful = OK Replication Failed = KO

    Step 24: For DB data. In Working Sheet D Colum “Today” in ME DB View we’ll have to put a formula. =VLOOKUP(Select A2 ,in automated report select data from B to J,9,0) Sample formula =VLOOKUP([@[Filespace Name]],'[ME_Infra_Operational_I2BCK_DLY_audit_logs_2022-08-01.xlsx]SLA'!\(B:\)J,9,0)

    Step 25: Once the formula us applied value will appear on D Colum working Sheet.

    Step 26: Apply filter and Select Replication successful. In “S” column mention “OK “ if we have any Replication failed values select that from the filter and & mention “S” Colum. For N/A values we should leave that blank in “S” Colum. Once we fill the data in S Colum. Delete the data from D column.

    Step 27: For VM data. In Working Sheet D Colum “Today” in VM View we’ll have to put a formula.

    =VLOOKUP(Select A2 ,in automated report select data from D to J,7,0)

    Sample formula

    =VLOOKUP([@[Filespace Name]],'[ME_Infra_Operational_I2BCK_DLY_audit_logs_2022-08-01.xlsx]SLA'!\(D:\)J,7,0))

    Step 28: Apply filter and Select Replication successful. In “S” column mention “OK “ if we have any Replication failed values select that from the filter and & mention “S” Colum. For N/A values we should leave that blank in “S” Colum. Once we fill the data in S Colum. Delete the data from D column.

    Step 29: Entering Value in DB and VM for region we are working on in working sheet. Move to DB tab. Refresh the sheet

    Step 30: Post refreshing do the right click and select Show field list. In Pivot table fields select the date for which we are working on. Once is selected check if value is appearing in main sheet. Repeat the same steps for VM as well.

    Step 31: Validate the data is accurate in Summary tab of working sheet.

    Step 32: Repeat the above steps for all the regions.

    Step 33: Uploading the file to ISC portal.

    Step 34: ISC portal link - https://www.ibm.com/services/connect/portal/

    Step 35: Once logon to ISC portal click on document. Click on Access Documents and other collaborative features.

    Step 36: Once we click on Access Documents and other collaborative features. It will take you new page. Select Enterprise report.

    Step 37: Select Enterprise I2BCK_Reports.

    Step 38: Select Daily.

    Step 39: Click on Add. Select new upload

    Step 40: Browse a file and upload the file.

    Step 41: Post upload we’ll have send email to following email id’s.
